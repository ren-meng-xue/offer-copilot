# 命名

## skill 目录名

目录名尽量简单，优先用：

- `product`
- `tech`
- `deploy`
- `resume`
- `interview`

如果必须带项目名，再加前缀：

- `offerpilot-product`
- `offerpilot-tech`

尽量少用下面这类名字：

- `xxx-context`
- `xxx-workflow-engine`
- `xxx-orchestrator`
- `xxx-recommendation`

原因很简单：太像模板生成，不像真实项目里的文件。

## references 文件名

文件名优先直接表达内容：

- `prd.md`
- `mvp.md`
- `user-flow.md`
- `tech-stack.md`
- `project-structure.md`
- `deploy.md`

不要为了“完整”去起很长的名字。
