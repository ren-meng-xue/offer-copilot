# 规则

## 放置位置

项目自定义 skill 统一放在：

```text
skills/<skill-name>/
```

不要放到系统目录里，也不要混到业务代码目录里。

## 基本结构

每个 skill 至少包含：

```text
<skill-name>/
├── SKILL.md
└── references/
```

如果内容很少，也可以先只有 `SKILL.md`。

## 写法要求

- `SKILL.md` 负责说明这个 skill 是干什么的
- `SKILL.md` 负责说明什么时候用、先看什么
- 详细资料放进 `references/`
- 不要把整份文档全部塞进 `SKILL.md`

## 风格要求

- 句子写短
- 少写空话
- 少写方法论口号
- 不用过多“智能化”“能力编排”“上下文感知”这类词
- 能直接写“做什么”，就不要写“适用于处理某类场景”
