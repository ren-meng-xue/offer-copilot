# 用户流程

## 主链路

`JD 输入 -> 岗位分析 -> 简历上传 -> 匹配评分 -> 简历优化 -> 面试准备 -> 投递`

## 产品上的理解

- `JD 输入` 是主入口，因为这个产品天然是岗位导向的
- `岗位分析` 用来建立信任，让用户先看到系统确实理解这个岗位
- `简历上传` 是把用户经历和目标岗位接起来
- `匹配评分` 用来制造紧迫感，也用来找缺口
- `简历优化` 要输出可以直接使用或继续编辑的结果
- `面试准备` 用来延长价值链，也更容易承接付费

## 设计含义

- 产品应当有引导感，不要做成空白聊天框
- 每一步最好都产出一个可查看的中间结果
- 评分不能裸给，必须带解释
- 生成内容要允许用户继续编辑
