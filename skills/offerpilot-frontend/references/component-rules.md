# 组件规则

## 基本原则

OfferPilot 前端默认使用 `shadcn/ui` 作为基础组件底座。

采用方式：

- 先初始化 `shadcn/ui`
- 用什么组件就 add 什么组件
- 组件代码进入项目后，可以按项目风格继续改

这不是黑盒组件库。

## 组件分层

- `components/ui/`
  放基础 UI 组件
  例如：
  - `button`
  - `input`
  - `card`
  - `dialog`
  - `tabs`

- `components/`
  放项目级通用组件
  例如：
  - 页面区块容器
  - 结果卡片
  - 状态标签
  - 空状态

- `features/*`
  放业务相关组件
  例如：
  - 登录表单
  - JD 输入面板
  - 匹配评分卡组

## 使用建议

- 优先复用 `ui` 基础组件，不要每页都手写一套按钮和输入框
- 页面里出现两次以上的业务区块，就考虑抽组件
- 不要把所有内容都塞进一个超大的页面文件
- 组件命名优先直接表达用途

## 当前阶段推荐优先安装的组件

- `button`
- `input`
- `label`
- `card`
- `form`
- `tabs`
- `badge`
- `dialog`
- `textarea`

## 当前阶段不建议

- 一开始就做完整设计系统
- 为了复用而过度抽象
- 在页面里堆大量重复 Tailwind 类名不整理
- 同一个语义做出多套风格不一致的组件
