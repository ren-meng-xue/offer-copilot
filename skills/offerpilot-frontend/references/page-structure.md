# 页面结构

## 主链路

OfferPilot 前端默认围绕这条主链路组织：

`JD 输入 -> 岗位分析 -> 简历上传 -> 匹配评分 -> 简历优化 -> 面试准备`

产品上不要做成一个空白聊天框。

每一步都应该有：

- 清晰的入口
- 明确的中间结果
- 可以继续下一步的动作

## 第一版建议页面

- `/`
  首页
  负责介绍产品价值、展示主流程，并引导用户进入登录或开始体验

- `/login`
  登录页
  负责邮箱密码登录

- `/register`
  注册页
  负责创建账号

- `/workspace`
  登录后工作台入口
  负责承接用户当前的目标岗位、简历和最近任务

- `/workspace/jobs/new`
  JD 输入页
  负责录入岗位描述或粘贴 JD

- `/workspace/jobs/[jobId]`
  岗位分析页
  负责展示关键词、要求、隐含预期和分析结果

- `/workspace/jobs/[jobId]/resume`
  简历上传页
  负责上传简历并展示解析状态

- `/workspace/jobs/[jobId]/analysis`
  匹配评分页
  负责展示分数、缺口和解释

- `/workspace/jobs/[jobId]/optimize`
  简历优化页
  负责展示 AI 改写结果，并支持继续编辑

- `/workspace/jobs/[jobId]/interview`
  面试准备页
  负责展示高频问题、追问和模拟面试入口

## 第一版不急着做的页面

- 复杂的个人设置中心
- 多主题切换
- 重型后台配置页
- 过细的运营页
- 泛化的聊天工作台

## 页面组织原则

- 首页负责引导，不负责承接全部功能
- 登录后页面优先按主链路推进
- 页面命名尽量直接，避免抽象词
- 一个页面优先只解决一个主要任务
