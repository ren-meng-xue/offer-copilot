# 环境变量约定

## 当前建议

- 前端和后端分开管理各自环境变量
- 所有需要提交给团队参考的变量，都提供示例文件
- 敏感信息不要直接写进仓库

## 文件建议

- 前端：`frontend/.env.example`
- 后端：`backend/.env.example`
- 如果后面有统一部署说明，可以在 `deploy/` 下补充环境说明文档

## 命名原则

- 变量名尽量直接表达用途
- 按服务边界拆分，不要混在一起
- 同一个值如果前后端都要用，要明确谁负责生成、谁负责消费

## 当前常见变量

- OpenAI API Key
- 数据库连接
- Redis 连接
- 对象存储配置
- 前端调用后端的 API 地址
