# 部署边界

## `deploy/` 是什么

`deploy/` 是项目的部署和运维配置区。

它不是单独的业务服务，也不是一个独立产品模块。它的作用是把发布和基础设施相关的东西单独收起来。

## 常见内容

- `Dockerfile`
- `docker-compose.yml`
- nginx 或其他代理配置
- `.env.example` 这类环境模板
- 发布脚本
- 云平台部署清单

## 实际理解

- `frontend/` 负责前端应用
- `backend/` 负责 API 和异步任务
- `deploy/` 负责把这些服务在开发或生产环境里跑起来

## 什么时候可以先保持很轻

如果前后端都直接托管到平台，`deploy/` 可以先很轻，只放环境模板或部署说明。

## 什么时候会变重要

下面这些场景一出现，`deploy/` 的价值就会上来：

- Docker 部署
- 自建服务器
- 反向代理
- 自定义域名路由
- HTTPS 终止
- CI/CD 自动发布
