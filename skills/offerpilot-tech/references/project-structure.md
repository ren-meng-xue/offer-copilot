# 项目结构

## 顶层目录

```text
offer-copilot/
├── frontend/
├── backend/
└── deploy/
```

## 目录职责

### `frontend/`

- 前端应用代码
- 页面和交互逻辑
- 前端构建配置
- 前端依赖管理

### `backend/`

- API 服务
- 业务逻辑
- 数据访问
- AI 处理编排
- 后端依赖管理

### `deploy/`

- 部署配置
- 容器化文件
- 反向代理配置
- 环境变量模板
- 部署脚本
- 和 CI/CD 发布有关的文件

## 当前规则

现阶段先保持这三个顶层目录即可。MVP 还没落地前，不要把仓库设计得太重。
